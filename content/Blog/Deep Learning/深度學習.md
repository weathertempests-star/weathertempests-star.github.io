# 深度學習：從課堂到理解的重建之旅

## 動機

這學期在陽明交大修習深度學習，課程橫跨三位老師的專業領域。
其中謝秉鈞老師的演算法課是大學時的記憶之一，痛並快樂著的那種。

藉期末複習的機會，把這學期學到的東西整理成一系列文章。
目的有兩個：

一是分享，讓有需要的人少走一些彎路；二是逼自己把東西真的搞懂，因為寫得出來才算懂。

---

## 這個系列在解決什麼問題？

修課到一半，我發現一件事：

~~(多數人)~~ 我對 Deep Learning 的理解是高度碎片化的。

我們大概知道：

- Transformer 很重要
- Attention 是核心機制
- CNN 適合影像
- Diffusion Model 能生成高品質圖片

但不一定真正理解：

- 這些模型究竟在解決什麼問題？
- 不同架構之間是如何演化的？
- 為什麼某些方法逐漸被淘汰？
- 訓練時實際發生了什麼？
- 數學推導背後的直覺是什麼？

所以這個系列的核心主軸只有一個：

> 把碎片化的知識，重新組織成一套可以回頭看、重新理解的系統。

---

## 這個系列會怎麼寫？

每篇文章都會圍繞一條主線進行：

```
Problem → Failure → Intuition → Formulation → Optimization → Engineering
```

也就是說，在談一個模型之前，先把它要解決的問題說清楚；再解釋舊方法為什麼不夠用；然後才進入直覺、公式與實作。

具體來說，每篇文章會盡量包含：

- 問題從哪裡來、動機是什麼
- 模型框架背後的物理或幾何意義
- 舊方法的失敗在哪裡
- 新架構如何改善
- 數學如何描述這些行為
- 實作上的細節

## 模型脈絡

這個系列真正想串起來的，不是單一模型，而是幾條互相呼應的演化線：

- **表示學習**：MLP → CNN / RNN → Transformer
- **潛變數建模**：PCA → PPCA → Autoencoder → VAE
- **生成模型**：density estimation → Flow → Diffusion；另一條是 VAE / GAN / WGAN 的對抗式與變分式路線
- **決策學習**：MDP → Value-based → Policy-based → Model-based

這些路線之間的共同點是：都在處理「資料怎麼被表示、怎麼被預測、怎麼被生成、怎麼被用來決策」這四件事。
