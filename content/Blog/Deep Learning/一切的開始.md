# 第一堂課：機器學習與深度學習到底在學什麼？

> 機器學習的核心是用資料找規律；
> 深度學習則把「如何表示資料」的責任交給多層可訓練函數。
> 本文以直觀例子與機率視角串起概念，說明為何 loss、模型與假設空間實為同一問題的不同面向。

## 這篇文章想回答什麼問題？

這堂課真正想回答的不是「模型叫什麼名字」，而是下面三個問題：

1. 機器學習到底在做什麼？
2. 為什麼我們需要 feature representation $\phi(x)$？
3. 深度學習和傳統機器學習，差別到底在哪裡？

如果把這堂課濃縮成一句話，就是：

> 機器學習是在資料中找規律；深度學習則是讓模型自己學會怎麼表示資料。

---

## 為什麼舊方法不夠好？從一個例子講起

假設我們手上有 MRI 影像 $x$，目標是預測病人是否健康，標籤是 $t$。在監督式學習裡，我們會拿大量 $(x, t)$ 配對資料來訓練模型，讓模型從輸入推測輸出。

但這裡有一個關鍵問題：

> 模型不一定適合直接吃原始資料。

原因很直接：原始資料通常太高維、太雜亂，也不一定含有模型容易利用的結構。於是我們常常先把 $x$ 轉成 representation：

$$\phi(x)$$

然後再把這個 representation 丟給模型做預測。換句話說，模型真正看到的不是原始資料本身，而是我們幫它整理過的特徵。

這就是傳統機器學習最核心的地方：**feature engineering 很重要**。

---

## 核心想法

### 機器學習是在找一個函數

從這堂課的角度來看，機器學習的目標可以很直接地寫成：

$$y = f_w(\phi(x))$$

其中 $w$ 是模型參數，$\phi(x)$ 是我們給模型的特徵表示。

如果是二元分類，我們通常會接上一個 sigmoid：

$$y = \sigma(w^T \phi(x)), \quad \sigma(s)=\frac{1}{1+e^{-s}}$$

這樣輸出就可以被解讀成介於 0 和 1 之間的值，進一步對應成機率。

課堂上的重點不是這個公式本身，而是它背後的意思：

> 我們想找一個函數，去逼近真實世界中的標籤生成規律。

### Mitchell 的 T、P、E 定義

課堂也引用了 Mitchell 的觀點：如果一個程式在任務 $T$ 上的效能 $P$，會隨著經驗 $E$ 增加而提升，我們就說它在學習。

套到 linear regression：

- 任務 $T$：給定輸入 $x$，預測輸出 $y$
- 經驗 $E$：使用訓練資料去調整參數 $w$
- 效能 $P$：在測試資料上看預測誤差有多小

這裡最重要的是最後一項。真正有價值的不是訓練集表現，而是**對沒看過的資料也能表現好**，也就是 generalization。

---

## 機器學習與深度學習的差別

深度學習其實仍然是機器學習，只是它把「怎麼表示資料」這件事也交給模型自己學。

### 傳統機器學習

傳統流程可以想成：

$$\text{原始資料 } x \;\to\; \text{人工設計特徵 } \phi(x) \;\to\; \text{線性模型 } w^T\phi(x) \;\to\; \text{預測結果}$$

也就是說，模型本體通常不負責處理原始資料結構，而是依賴我們先把資料整理成好用的 representation。

### 深度學習

深度學習的差別在於，特徵不再靠人手工設計，而是靠多層函數自己學出來：

$$f_{w,\theta_n,\dots,\theta_1}(x) = \sigma\bigl(w^T\phi_{\theta_n}(\cdots\phi_{\theta_1}(x))\bigr)$$

這個式子看起來很長，但概念其實很單純：

- 第一層先抽出簡單模式
- 第二層把簡單模式組合成更抽象的模式
- 更高層再把抽象模式組成更高階概念

這就是課堂上說的 **hierarchy of concepts**。以一個 FeedForward Network 辨識圖片為例：

- 第一層可能學到邊緣
- 第二層可能學到角落與輪廓
- 更高層可能學到物件局部
- 最後輸出層學到整體物件類別

這也是為什麼深度學習會讓人覺得強：它不只是把輸入映射到輸出，而是在學一整套逐層抽象化的表示方式。

---

## 數學推導：線性迴歸其實就是最小平方法

舉一個經典的例子：linear regression（不做分類，而是直接預測連續值）。

模型寫成：

$$\hat{y} = w^T\phi(x)$$

對整個訓練集來說，把每筆資料的特徵排成設計矩陣 $\Phi$，就得到：

$$\hat{y} = \Phi w$$

接著用 mean squared error 當損失：

$$\|\,y - \Phi w\,\|_2^2$$

這就是最小平方法問題。因為損失對 $w$ 來說是 quadratic function，所以可以直接得到 closed-form solution：

$$w^* = (\Phi^T\Phi)^{-1}\Phi^T y$$

這一點很重要，因為它說明了線性迴歸和很多深度學習模型最大的差別：

- 線性迴歸在特定設定下，可以一次解出最佳參數
- 深度學習通常沒有這種漂亮的封閉解，所以要靠 gradient-based learning

### 加上 bias 之後怎麼辦？

如果模型變成 $\hat{y} = w^T\phi(x) + b$，其實仍然可以寫成內積形式。只要把特徵和參數一起擴充：

$$\bar{\phi}(x) = \begin{bmatrix} \phi(x) \\ 1 \end{bmatrix}, \quad \bar{w} = \begin{bmatrix} w \\ b \end{bmatrix} \implies \hat{y} = \bar{w}^T\bar{\phi}(x)$$

也就是說，bias 並沒有讓模型本質改變，只是多加了一個常數維度。

### 特徵可以長什麼樣子？

$\phi(x)$ 不一定要是線性的。你可以選 $[x^2, x, 1]^T$，也可以選 $[\sin(x), \cos(x), 1]^T$，甚至可以選 Gaussian basis functions。重點是：

> 只要特徵選得對，模型就能用同一套最小平方法去擬合不同形狀的函數。

## 參數估計：從 MLE 到 KL-divergence

前面我們一直在說「把訓練誤差變小」，但如果從統計學的角度來看，這其實是在做**參數估計**。視角就會從幾何和代數轉為**機率論**。

### 一個不公平的硬幣：直覺從這裡開始

在進入公式之前，先用一個具體例子把直覺建立起來。

假設你手上有一枚**不公平的硬幣**，它出現正面的機率是某個未知的 $\theta$。你現在想辦法估計 $\theta$。

你的做法很自然：丟 $m$ 次，觀察到 $k$ 次正面、$m-k$ 次反面。

那麼問題來了：光看這組資料，$\theta$ 應該是多少？

**從 likelihood 的角度想：** 如果 $\theta$ 真的是這個值，觀測到「$k$ 次正面、$m-k$ 次反面」的機率是多少？

$$L(\theta) = \theta^k (1-\theta)^{m-k}$$

我們想找的 $\theta$，就是讓這件事發生機率最大的那個值。這就是 **Maximum Likelihood Estimation（MLE）** 的核心思想。

#### 動手算一次

取 log（因為 log 是單調遞增，極值位置不變）：

$$\log L(\theta) = k \log \theta + (m-k) \log(1-\theta)$$

對 $\theta$ 微分並令其為零：

$$\frac{d}{d\theta} \log L(\theta) = \frac{k}{\theta} - \frac{m-k}{1-\theta} = 0$$

解出來：

$$k(1-\theta) = (m-k)\theta \implies k = m\theta \implies \boxed{\theta^* = \frac{k}{m}}$$

結果很直覺：**最大概似估計就是正面出現的頻率**。丟 100 次出現 63 次正面，$\theta^* = 0.63$。

> **Insight：** MLE 不是什麼神秘的技術，它做的事情就是「找一個參數，讓我觀測到這組資料的機率最大」。在這個簡單的例子裡，答案就是樣本頻率。

---

### 推廣到一般情況

假設我們有一組**獨立同分佈 (i.i.d.)** 的資料 $x^{(1)}, x^{(2)}, \dots, x^{(m)}$，模型由參數 $\theta$ 控制。由於資料彼此獨立，likelihood 可以寫成：

$$L(\theta) = \prod_{i=1}^{m} p(x^{(i)} \mid \theta)$$

取對數：

$$\log L(\theta) = \sum_{i=1}^{m} \log p(x^{(i)} \mid \theta)$$

MLE 就是找讓這個式子最大的 $\theta$：

$$\theta^*_{\text{MLE}} = \arg\max_\theta \sum_{i=1}^{m} \log p(x^{(i)} \mid \theta)$$

---

### MLE 和 KL-divergence 的等價關係

這是第一堂課最重要的地方，也是很多人忽略的連結。

先換一個視角：如果把訓練資料看成來自某個**真實分佈** $p_{\text{data}}(x)$，而我們的模型是 $p_{\text{model}}(x;\theta)$，那最大化 log-likelihood 可以改寫成期望值的形式：

$$\max_\theta \;\mathbb{E}_{x \sim p_{\text{data}}} [\log p_{\text{model}}(x;\theta)]$$

（因為訓練集就是從 $p_{\text{data}}$ 抽樣出來的，所以樣本平均就是期望值的近似。）

#### KL-divergence 是什麼？

KL-divergence 量的是「兩個分佈有多不像」：

$$\text{KL}(p_{\text{data}} \,\|\, p_{\text{model}}) = \mathbb{E}_{x \sim p_{\text{data}}} \left[\log p_{\text{data}}(x) - \log p_{\text{model}}(x;\theta)\right]$$

把它展開：

$$= \underbrace{\mathbb{E}_{x \sim p_{\text{data}}} [\log p_{\text{data}}(x)]}_{\text{與 } \theta \text{ 無關，是常數}} \;-\; \mathbb{E}_{x \sim p_{\text{data}}} [\log p_{\text{model}}(x;\theta)]$$

最小化 KL-divergence，只需要動第二項（第一項跟 $\theta$ 無關）：

$$\min_\theta \;\text{KL}(p_{\text{data}} \,\|\, p_{\text{model}}) \;\iff\; \max_\theta \;\mathbb{E}_{x \sim p_{\text{data}}} [\log p_{\text{model}}(x;\theta)]$$

也就是說：

> **最大化 log-likelihood $\iff$ 最小化 $\text{KL}(p_{\text{data}} \| p_{\text{model}})$**

兩件事完全等價。

#### 回到不公平的硬幣驗證一下

真實分佈 $p_{\text{data}}$：$P(H) = p^*$，$P(T) = 1-p^*$（但我們不知道 $p^*$）

模型分佈 $p_{\text{model}}$：$P(H) = \theta$，$P(T) = 1-\theta$（我們要學 $\theta$）

KL-divergence：

$$\text{KL}(p_{\text{data}} \,\|\, p_{\text{model}}) = p^* \log\frac{p^*}{\theta} + (1-p^*)\log\frac{1-p^*}{1-\theta}$$

對 $\theta$ 取微分並令其為零：

$$\frac{\partial}{\partial \theta}\text{KL} = -\frac{p^*}{\theta} + \frac{1-p^*}{1-\theta} = 0 \implies \theta^* = p^*$$

最小化 KL 的最佳解就是讓模型分佈等於真實分佈。而我們用樣本估計 $p^*$ 時，得到的就是頻率 $k/m$，和 MLE 結果一模一樣。

> **Insight：** MLE 不是單純「讓資料機率最大」的技術手段，它背後做的事情是**讓模型分佈盡量貼近資料的真實分佈**。這才是它在深度學習裡為什麼重要的原因。

---

### 這件事和深度學習的連結

當我們訓練一個神經網路時，選擇不同的 loss function，本質上就是在選擇不同的機率模型，然後對那個模型做 MLE：

| 任務 | 機率模型假設 | 等價的 Loss Function |
|------|-------------|---------------------|
| 回歸 | $p(y \mid x) = \mathcal{N}(f_\theta(x), \sigma^2)$ | Mean Squared Error |
| 二元分類 | $p(y \mid x) = \text{Bernoulli}(\sigma(f_\theta(x)))$ | Binary Cross-Entropy |
| 多類別分類 | $p(y \mid x) = \text{Categorical}(\text{softmax}(f_\theta(x)))$ | Cross-Entropy |

所以當你在寫 `nn.CrossEntropyLoss()` 的時候，其實是在說：「我假設資料是從某個 Categorical 分佈生成的，我要用 MLE 來估計這個分佈的參數。」

Loss function 不是人工設計的懲罰項，而是機率模型的直接推論。

---

### MLE 和 MAP 的差別

如果我們再加上先驗分佈 $p(\theta)$，就會得到 MAP（Maximum A Posteriori）估計：

$$\theta_{\text{MAP}} = \arg\max_\theta \bigl(\log p(X \mid \theta) + \log p(\theta)\bigr)$$

這一項很關鍵，因為它跟 regularization 的關係非常直接。當你對參數加上懲罰項時，本質上就是在引入先驗，讓模型不要只盯著訓練資料。

具體來說：

- 如果先驗是 Gaussian（$\theta \sim \mathcal{N}(0, \sigma^2)$），$\log p(\theta) \propto -\|\theta\|^2$，對應 **L2 regularization（weight decay）**
- 如果先驗是 Laplace，對應 **L1 regularization**

也就是說：

- MLE 偏向「把資料解釋好」
- MAP 則是在「把資料解釋好」的同時，順便限制參數不要太極端

這也把前面的 generalization 和後面的 regularization 接起來了。

## 訓練時實際發生什麼？

### 最小化訓練誤差不等於真的學會

課堂一開始就提醒了一件事：

> 最小化訓練誤差，不等於真的學會。

如果我們只是在乎訓練資料上的誤差，那比較像 optimization。但 machine learning 真正關心的是：**模型對未見過的資料表現如何**。這就是 generalization。

我們假設資料來自某個未知分佈 $p_{\text{data}}(x, y)$，訓練只是從這個分佈抽樣的一小部分。模型如果只是記住訓練樣本，遇到新資料就會失敗。

### 模型容量與過擬合

當模型太弱時，會 underfitting；當模型太強、資料太少時，會 overfitting。

- **欠擬合**：模型太簡單，連訓練資料都學不好
- **過擬合**：模型太複雜，把訓練資料中的噪聲也一起記住了

這也是為什麼 regularization 會出現：我們願意多付一點訓練誤差，換取更好的泛化能力。

## 常見誤解

### 1. 深度學習不是另一種完全不同的東西

它本質上仍然是 machine learning。差別在於它把「特徵學習」也納入學習過程，而不是靠人工先寫好。

### 2. 模型不是只要複雜就好

模型太複雜不代表一定更準。若資料不足，複雜模型反而更容易 overfit。

### 3. Loss function 不是你「選」的

在機率模型的框架下，loss function 是你對資料生成過程的假設推導出來的，而不是任意挑一個看起來合理的懲罰項。

### 4. 線性模型也能做出非線性效果

只要 feature mapping 夠強，線性模型一樣可以擬合很複雜的函數。真正決定表達能力的，往往是 feature representation，而不只是最後那一層的形式。

---

## 比較

### 傳統機器學習

重點在人工設計特徵。優點是簡單、可解釋、理論清楚。缺點是你得先知道該怎麼做特徵。

### 深度學習

重點在自動學特徵。優點是可以處理複雜資料，尤其是影像、語音、文字。缺點是參數多、訓練難、通常沒有 closed-form solution。

---

## 總結

1. 機器學習的目標是找一個函數去逼近資料背後的規律；線性迴歸展示了最基本的形式，而且在特定設定下能直接求解。

2. MLE 的直覺是「讓觀測到這組資料的機率最大」，以不公平硬幣為例，最佳解就是樣本頻率 $k/m$。

3. **最大化 log-likelihood $\iff$ 最小化 KL-divergence**，也就是說訓練目標不只是降低誤差，而是讓模型分佈盡量貼近真實資料分佈。

4. 深度學習裡各種 loss function（MSE、Cross-Entropy）都是在特定機率模型假設下做 MLE 的直接推論，不是人工設計的懲罰項。

5. 加入先驗就變成 MAP，對應 regularization；這把 generalization 和訓練目標串接在一起。
